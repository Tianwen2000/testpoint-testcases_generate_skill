# 音创AI V1.8.0 AI 辅助测试设计样例

本项目以 **音创AI/Melod AI V1.8.0** 版本迭代测试为例，记录一次使用 AI 辅助完成测试分析、测试点拆解、测试用例生成与覆盖追溯的实践过程。

它不是业务代码仓库，而是一套测试设计工作样例：通过沉淀 prompt、skill、测试点结构和生成脚本，展示如何把一次版本迭代中的需求材料转化为可评审、可执行、可追溯的测试产物。

## 项目目标

- 以 V1.8.0 迭代需求为样本，展示 AI 如何辅助测试设计。
- 沉淀可复用的测试分析、测试点生成、测试用例生成 skill。
- 用结构化脚本把测试点导图和测试用例表落成文件。
- 保留覆盖矩阵、叶子节点映射、待确认事项，避免 AI 只生成看似完整但不可追溯的用例。
- 总结在 PRD、Figma、历史版本需求混合输入时的注意事项。

## 工作流

推荐按以下顺序使用本项目中的材料：

1. **需求分析**

   使用 `需求分析通用skill.md`，以上一版本需求为基线、以本版本需求为目标，先做增量分析和受影响回归分析。

2. **测试点拆解**

   使用 `xmind测试点生成器skill.md`，基于 PRD、Figma、需求分析简报和补充澄清，生成模块化 XMind 测试点。

3. **测试用例生成**

   使用 `测试用例生成器skill.md`，基于 XMind 测试点、PRD、Figma 和需求分析结果，生成可执行测试用例。

4. **脚本落地**

   使用 `generate_v180_testpoint_xmind.py` 生成 XMind 文件，使用 `generate_v180_testcases_xlsx.py` 生成 Excel 测试用例和追溯表。

5. **人工复核**

   重点复核待确认事项、Figma 与 PRD 冲突、商业化分流、支付回跳、异步任务状态、权限和异常流程。

| 文件 | 作用 |
| --- | --- |
| `需求分析通用skill.md` | 将 PRD/需求文档转成面向测试设计的需求分析简报，强调规则、缺口、风险和追溯编号。 |
| `xmind测试点生成器skill.md` | 指导 AI 生成 XMind 8 旧版兼容测试点导图，强调模块声明、链路型拆解和结构校验。 |
| `测试用例生成器skill.md` | 指导 AI 基于 XMind、PRD、Figma 生成中文测试用例，并同步输出覆盖矩阵、叶子节点映射和待确认项。 |
| `generate_v180_testpoint_xmind.py` | 内置 V1.8.0 的测试点树，生成首页、创作页、我的三个模块的 `.xmind` 文件。 |
| `generate_v180_testcases_xlsx.py` | 基于测试点树生成 Excel 测试用例、核心测试点覆盖矩阵、XMind 叶子节点映射、待确认事项和统计表。 |
| `generate_v180_testcases_xlsx.mjs` | 使用 `@oai/artifact-tool` 的 Excel 生成版本，适合在具备对应运行时的环境中参考。 |
| `figma的mcp配置.json` | Figma MCP 相关配置记录。 |
| `docs/迭代项目额外的prompt` | 迭代版本分析时的补充 prompt，强调以上版本为基线、本版本为目标版本。 |
| `docs/注意事项` | 记录 Figma 与需求混合输入时 AI 容易混淆的问题。 |

## 运行方式

### 1. 生成 XMind 测试点

```bash
python3 generate_v180_testpoint_xmind.py
```

当前脚本会生成 3 个模块导图：

- `../testpoints_xmind/音创AI_MelodAI_V1.8.0_首页_测试点.xmind`
- `../testpoints_xmind/音创AI_MelodAI_V1.8.0_创作页_测试点.xmind`
- `../testpoints_xmind/音创AI_MelodAI_V1.8.0_我的_测试点.xmind`

本次验证结果：

| 模块 | 节点数 |
| --- | ---: |
| 首页 | 86 |
| 创作页 | 128 |
| 我的 | 155 |

### 2. 生成 Excel 测试用例

```bash
python3 generate_v180_testcases_xlsx.py
```

依赖：

```bash
pip install openpyxl
```

## 样例覆盖范围

本样例覆盖 V1.8.0 迭代中三个主要模块：

- **首页**：新版首页、订阅页触发、首页分轨入口、上传后状态页、会员/非会员分流、支付回跳、分轨结果反馈、商业化扣费、异常与兼容。
- **创作页**：灵感创作、高级创作、模型选择、音频参考、风格标签、风格优化、高级参数、AI 生成任务、异常状态和数据带入。
- **我的**：专辑、作品播放页、下载、分享、删除、分轨音乐名称展示、商业化能力边界和回归风险。

## 复用到其他迭代

如果要把这套方式复用到下一个版本，可以按下面步骤改造：

1. 准备上版本需求文档、本版本需求文档、Figma 设计稿和补充澄清记录。
2. 使用 `docs/迭代项目额外的prompt` 指定“基线版本”和“目标版本”，先做增量分析。
3. 使用 `需求分析通用skill.md` 输出需求分析简报，明确继承规则、新增规则、变更规则、删除/失效规则和回归关注点。
4. 在使用 `xmind测试点生成器skill.md` 前，手工声明本次模块名，避免 AI 擅自把所有页面都当作模块。
5. 根据 AI 输出结果更新 `generate_v180_testpoint_xmind.py` 中的 `TREES`。
6. 重新运行 XMind 和 Excel 生成脚本。
7. 人工复核待确认事项，不要把未定义规则直接写进正式预期结果。

## 注意事项

- AI 生成结果必须经过人工复核，尤其是商业化、扣费、权益、支付回跳、任务失败、前后台恢复等高风险场景。
- PRD 与 Figma 不一致时，不要让 AI 直接混合推断。建议先输出冲突和待确认项，再进入测试点或用例生成。
- 迭代项目不建议做无意义的全量重写。应以上版本为基线，重点分析本次增量和受影响回归。
- XMind 是用例生成的主骨架。正式用例必须能追溯到 XMind 叶子节点或明确说明未独立成例原因。
- 待确认事项只记录需求缺口和冲突，不应污染正式测试用例的预期结果。
- `generate_v180_testcases_xlsx.mjs` 包含特定运行时路径和 `@oai/artifact-tool` 依赖，常规本地执行优先使用 Python 版本。

## 项目价值

这套样例的重点不是“让 AI 一次性写完所有测试用例”，而是把测试设计拆成可控步骤：

- 先理解需求和边界；
- 再拆测试点；
- 再生成用例；
- 最后用覆盖矩阵和映射表证明没有明显漏测。

通过这种方式，AI 更适合作为测试设计助手，而不是替代测试人员判断需求风险、版本边界和最终测试范围。
